Design für Alle

Adaptive/Responsive Design

  • Flexibles Design: Unterschiedliche Darstellung je nach Ausgabemedium
  • Notwendig aufgrund der Vielfalt an Geräten (Touchscreens, Fernseher, Smartphones, zukünftige Geräte)
  • Beispielsweise: Auf Mobilgerät Informationen in Blöcke untereinander darstellen, auf Desktop nebeneinander

Unterschiedliche Herangehensweisen:

AdaptiveResponsive
StrategieEntwicklung für bestimmte AusgabegeräteEntwicklung für alle Gerätetypen
Bedeutungan Ausgabemedium angepasstauf Ausgabemedium reagierend
Layoutggf. mehrere unterschiedliche Layoutsein flexibles Layout für alle Geräte
Inhalteggf. unterschiedliche Inhaltedieselben Inhalte auf allen Geräten
VorteileDatenmenge und Latenz kann berücksichtigt werdenUnbekannte oder neue Geräte werden unterstützt
EntwicklungSeiten werden separat für verschiedene Geräte entwickeltMobile-First-Ansatz oft vorteilhaft

Adaptiv vs. Responsive : Entscheidung basierend auf Nutzeranalyse

Barrierefreiheit

Barrierefreiheit stellt sicher, dass Webinhalte für alle Menschen zugänglich sind, unabhängig von körperlichen oder technischen Einschränkungen. Das W3C hat die [Web Content Accessibility Guidelines (WCAG) 2.0](W3C: Web Content Accessibility Guidelines (WCAG) 2.0 formuliert, die in vier Kategorien unterteilt sind:

  • Wahrnehmbar, z. B. Textalternativen für Nicht-Text Inhalte; Alternativen für zeitbasierte Medien
  • Bedienbar, z. B. alle Funktionalitäten von Tastatur aus verfügbar; ausreichend Zeit zum lesen und benutzen lassen
  • Verständlich, z. B. Textinhalte lesbar und verständlich; vorhersehbares Aussehen und Funktion
  • Robust, z. B. zuverlässig interpretierbar von verschiedenen Benutzeragenten (z. B. Browser inkl. Assistenzsystem)

Nach Hellbusch und Probiesch gibt es sieben zentrale Punkte für barrierefreies Webdesign :

  1. Textorientierung: Alternativtexte für Nicht-Text-Inhalte.
  2. Kontraste und Farben: Für ausreichende Lesbarkeit sorgen.
  3. Skalierbarkeit: Flexible Layouts und skalierbare Schriftgrößen.
  4. Linearisierbarkeit: Inhalte müssen auch ohne CSS in einer logischen Reihenfolge erscheinen.
  5. Geräteunabhängigkeit: Inhalte sollten auch mit der Tastatur navigierbar sein.
  6. Verständlichkeit und Navigation: Inhalte und Navigation müssen klar und verständlich sein.
  7. Strukturierte Inhalte: Korrekte Verwendung von HTML-Strukturelementen.

Hinweis

Durch Browser Einstellungen wie “Surfen ohne Bilder”, oder Browsererweiterungen wie “Web Developer Extension” kann Barrierefreiheit getestet werden.

Testing und Kompatibilität:

  • Es ist wichtig, Websites in verschiedenen Browsern zu testen, um sicherzustellen, dass alle Zielgruppen abgedeckt werden.
