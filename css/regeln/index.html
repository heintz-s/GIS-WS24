Regeln festlegen

CSS Regeln

Aufbau

Eine CSS-Regel besteht aus zwei wesentlichen Teilen:

  • Selektor: Bestimmt, auf welche Elemente die Regel angewendet wird.
  • Deklaration:
    • Eigenschaft: Definiert, welcher Aspekt des Elements angepasst werden soll.
    • Wert: Gibt an, wie die Eigenschaft umgesetzt wird.

Kommentare

Kommentare werden in CSS mit /* und */ umschlossen: /* Kommentar */

Regeln zusammenfassen

Regeln mit dem gleichen Selector können im Anweisungsblock zusammengefasst werden.

CSS Selektoren

Es gibt verschiedene Arten von Selektoren, um ein oder mehrere Elemente auszuwählen:

Tipp

Das erste Zeichen des Selektors legt die Art des Selektors fest:

  • ohne = Typeselektor
  • . = Klassenselektor häufigster
  • # = id-Selektor
  • [ = Attributselektor; endet mit ]

Typselektor: tagName

Wählt alle HTML-Elemente mit dem angegebenen Tag aus.

Hinweis

der Universalselektor * selektiert alle Elemente

Klassenselektor: .klassenName

Wählt alle Elemente mit dem angegebenen Klassennamen aus, also alle Elemente mit class=klassenName als Attribut.

Tipp

Elemente können mehrere Klassennamen haben. Diese werden mit Leerzeichen getrennt: class="name1 name2 name3"

ID-Selektor: #idWert

Wählt das Element mit der entsprechenden id aus, also das Element mitid=idWert als Attribut.

Hinweis

IDs sollten eindeutig sein, also nur einmal in einer HTML Seite vorkommen.

Attributselektoren: [attributName]

Wählt Elemente basierend auf einem bestimmten Attribut aus, also alle Elemente, die ein Attribut mit dem angegebenen attributName besitzen.

Hinweis

Kann weiter spezifiziert werden über die Angabe eines Wertes, mit [attributName = wert]. Es können auch Teilübereinstimmungen mit dem Wert geprüft werden (siehe https://wiki.selfhtml.org/wiki/CSS/Tutorials/Selektoren/einfacher_Selektor#Attributselektor ).

CSS Selektoren kombinieren

Um die Auswahl von Elementen weiter zu präzisieren und um Regeln effizienter zusammenzufassen, können CSS-Selektoren auf verschiedene Arten kombiniert werden.

Liste von Selektoren: selektor1, selektor2

  • Mit Komma separierte Aufzählung von Selektoren
  • Wendet den Anweisungsblock auf alle angegebenen Selektoren an.

Verbundene Selektoren: selektor1selektor2

  • Ohne Leerzeichen direkt hintereinander geschriebene Selektoren
  • Wählt Elemente, die alle Selektoren erfüllen.

Hinweis

Ist ein Elementselektor enthalten, steht dieser immer vorne.

Mit Kombinatoren verbundene Selektoren: selektor1 selektor2

  • Kombinatoren sind Zeichen, mit denen Selektoren verbunden werden: Liste möglicher Kombinatoren
  • Selektiert selector2 Elemente, welche in passender Beziehung zu selector1 stehen.
  • Leerzeichen als Kombinator: Wählt selektor2-Elemente, die Nachfahren von selektor1 sind (im HTML Baum).

Spiel zum Lernen

Kaskade

  • CSS folgt dem Prinzip der Kaskade, bei dem mehrere Regeln auf das gleiche Element angewendet werden können.
  • Die Reihenfolge und Priorität der Regeln bestimmt, welche letztendlich zum Tragen kommen.
  • Die Regeln werden also in eine Abfolge gebracht und nacheinander angewendet.
  • Gleiche Eigenschaften werden dabei jeweils überschrieben.

Wasserfall Kaskade: fällt über mehrere Stufen

Tipp

Entwicklungsumgebung im Browser öffnen und Auflistung der CSS Regeln im “Styles” Tab betrachten. Überschriebene Deklarationen sind durchgestrichen.

Priorität in CSS

Die Priorität der Regeln wird in der folgenden Reihenfolge bewertet (von hoch nach niedrig):

  1. !important-Deklarationen nur in Ausnahmefällen nutzen
  2. Inline-Styles über style-Attribut direkt im HTML
  3. Anzahl der ID-Selektoren
  4. Anzahl der Klassenselektoren und Attributselektoren
  5. Anzahl der Typselektoren

Tipp

Bei gleicher Priorität wird nach Reihenfolge im CSS Dokument entschieden. Bei mehreren eingebundenen Dokumenten auch nach Reihenfolge der Dokumente.

Hinweis

Browser geben ein einfaches Defaul-Styling vor, den “user agent style”. Dieser wird immer am niedrigsten priorisiert.

Hinweis

Zeigt eine Regel keine Wirkung -> Priorität prüfen!

Vererbung

Einige CSS-Eigenschaften werden automatisch an Kindelemente vererbt, z. B. color oder font.

  • manche Eigenschaften können vererbt werden (z. B. color, font)[^element_ohne_vererbung]
  • Werte werden dann an die Kindelemente weitergegeben (bei Berechnung Ergebnis, nicht Rechnung)
  • Vererbung einer Eigenschaft mit folgenden Werten beeinflussen:
    • inherit erzwingt Vererbung
    • initial stellt Standardwert wieder her
    • unset bei vererbten Eigenschaften inherit, sonst initial

Hinweis

  • Vererbungsregeln können über Werte in Deklarationen angepasst werden (mit inherit, initial, unset).
  • Manche Elemente sind von der Vererbung ausgeschlossen (z. B. input).
