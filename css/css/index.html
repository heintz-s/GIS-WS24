CSS Grundlagen

CSS

  • Cascading Style Sheets: Stufenförmige Formatvorlagen
  • Formatierungssprache (Gestaltung, Design) für HTML, aber auch SVG und XML
  • Vorteil: Trennung des Inhalts von Struktur und Design

Beispiel:

CSS einbinden

  1. Direkt am HTML Element, über “style” Attribut für Element definieren

    • bezieht sich nur auf das Element
    • nicht für globale Styles geeignet (z. B. alle Überschriften gleich)
    • sollte nur im Ausnahmefall verwendet werden

  2. Im head Element, über style Element für einzelne Seite definieren

    • bezieht sich nur auf die Seite in der es eingebunden ist
    • nicht für seitenübergreifende Styles geeignet

  3. Im head Element, zentrale CSS Datei für einzelne Seite laden

    • kann in jeder Seite eingebunden werden
    • alle Styles zentral verwaltet
    • übliche Vorgehensweise

CSS Regeln

Jede Regel besteht aus:

  • Selector: Auswahl der Elemente, auf welche die Regel angewendet werden soll
  • Deklaration:
    • Eigenschaft: Welche Eigenschaft des Elements soll angepasst werden
    • Wert: Wie soll die Eigenschaft des Elements angepasst werden

Kommentare werden mit /* und */ umschlossen: /* Kommentar */

Regeln mit dem gleichen Selector können im Anweisungsblock zusammengefasst werden.

CSS Selektoren

Arten von Selektoren:

  1. Typselektor: tagName
  • Name eines Tags 1
  • selektiert alle Elemente mit diesem Tag
  1. ID-Selektor: #idWert
  • Wert des id Attributs eines Elements
  • vorangestelltes Hash
  • selektiert genau das Element mit der angegebenen ID
  • Hinweis: IDs sollten immer einmalig sein in einer HTML Seite
  1. Klassenselektor: .klassenName
  • Wert des class Attributs von Elementen
  • vorangestellter Punkt
  • selektiert alle Elemente mit diesem Klassennamen
  • Elemente können mehrere Klassennamen haben (mit Leerzeichen getrennt: class="name1 name2 name3")
  1. Attributselektoren: [attributName]
  • Name des Attributs von Elementen
  • umschlossen von eckigen Klammern
  • selektiert alle Elemente mit diesem Attribut
  • Hinweis: kann weiter spezifiziert werden über die Angabe eines Wertes: [attributName = wert] 2

CSS Selektoren kombinieren

  1. Liste von Selektoren: selektor1, selektor2
  • mit Komma separierte Aufzählung von Selektoren
  • nachfolgender Anweisungsblock auf alle angewendet
  1. Verbundene Selektoren: selektor1selektor2
  • ohne Leerzeichen hintereinander geschriebene Selektoren
  • selektiert Elemente, die alle verbundenen Selektoren erfüllen
  • Typselektor immer vorne (oder ohne)
  1. Mit Kombinatoren verbundene Selektoren: selektor1 selektor2
  • mit (Leer-)Zeichen verbundene Selektoren
  • selektiert selector2 Elemente, welche in passender Beziehung zu selector1 stehen
  • bei Leerzeichen muss das selektor2 Element ein selektor1 Element als Vorfahre im Baum haben
  • weitere Kombinatoren

Element Gestaltung

  • HTML Elemente können mit verschiedenen CSS Eigenschaften gestaltet werden
  • nicht jedes HTML Element hat jede CSS Eigenschaft (kann definiert werden, aber ohne Auswirkung)
  • Größe und Abstand von anderen Elementen wird durch eine Box definiert

Farben, Maßeinheiten, Schriftarten

Farbangaben:

  • Name, z. B. white, red, weitere; gut zum Testen, da leicht zu merken
  • RGB Hexwert: #rrggbb jeweils 00 - FF
  • RGB Funktion: rgb(r, g, b) oder mit Transparenz rbga(r, g, b, a) jeweils mit Werten 0 - 255 oder 0% - 100% (Transparenz: 0 - 1)
  • HSL Funktion: hsla(farbton, sättigung, helligkeit, deckkraft) mit Werten (0 - 360, 0% - 100%,0% - 100%, 0 - 1)
  • opacity: Transparenz eines Elements 0 - 1

W3 Empfehlung zur Verwendung, W3 Längen auf Englisch

Schriftarten:

  • CSS Font Stack: Liste von Schriftarten mit Komma getrennt; die erste vorhandene wird verwendet; z. B. Wunschschrift, Websichere Schrift, Schriftfamilie
  • Generische Schriftfamilie: serif, sans-serif, monospace, cursive, fantasy; immer als Fallback als letztes angeben -> Browser wählt dann Schriftart aus dieser Familie
  • Websichere Schriftarten: auf verschiedenen Systemen fast immer vorhanden; für Desktop-Systeme z. B. Arial, Georgia, Times New Roman, Courier New, Verdana; seit Mobile Nutzung schwieriger
  • Webfonts: Einbettung von Schriftarten seit CSS2 mit @font-face; selbst gehostet oder fremd gehostet (z. B. google fonts)
  • Ausblick: Variable fonts

Beispiel für Webfonts: https://fonts.google.com/specimen/Akronim#standard-styles)

Layout - Flussverhalten

Der Fluss entspricht dem Schreibfluss (lateinische Schrift: von links nach rechts und von oben nach unten)

displayFlussverhaltenz. B. Standard für HTML Element
inlineElemente horizontal in Schreibrichtung; hintereinanderem, strong, span, img, a
blockBox über komplette Breite des Elternelements; untereinanderp, div, table, h1
noneunsichtbar, ohne Lücke 5-

Umfließen von Elementen:

  • float: Element steht links (left) oder rechts (right) und wird von nachfolgenden Elementen umflossen
  • clear: umfließt kein Element auf der linken (left), rechten (right) oder beiden Seiten (both)

relativevon Fluss Position wegbewegen und dort Lücke lassen

absoluteam nähsten positionierten Elternelement ausgerichtet

Positionieren mit top, bottom, left, right, z-index

Komplexere Layouts

  • nicht mehr über Tabellen
  • Flex: display: flex | inline-flex; vertikale oder horizontale Positionierung von Elementen mit Einstellungen für Anordnung und Platzverteilung; Spiel zum Lernen
  • Grid: display: grid | inline-grid; Anordnung von Elementen in einem selbst definierten Gitter; Spiel zum Lernen
