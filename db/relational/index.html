Relationale Datenbanken

Datenbanken

Datenbanken sind strukturierte Systeme zur Verwaltung von Daten. Sie ermöglichen effizienten Zugriff, Speicherung und Manipulation großer Datenmengen.

Anforderungen an Datenbanken

Typische Anforderungen an moderne Datenbanken umfassen:

  • Verarbeitung großer Datenmengen: Optimale Verwaltung und Skalierung.
  • Schneller Zugriff: Minimale Latenzzeiten bei Abfragen.
  • Paralleler Zugriff: Unterstützung mehrerer gleichzeitiger Benutzer.
  • Konsistenz: Sicherstellung der Datenintegrität (= Korrektheit der Daten).
  • Sicherheit: Schutz vor unbefugtem Zugriff und Datenverlust.
Datenbankmanagementsystem (DBMS)

Ein Datenbankmanagementsystem (DBMS) ist Software, die die Verwaltung und den Zugriff auf Daten in einer Datenbank steuert.

CRUD-Operationen

Persistente (dauerhafte) Speicher bieten meist vier grundlegende Operationen zur Verwaltung der gespeicherten Daten:

  • Create (Erstellen): Hinzufügen neuer Datensätze.
  • Read (Lesen): Abrufen von Datensätzen.
  • Update (Bearbeiten): Ändern bestehender Daten.
  • Delete (Löschen): Entfernen von Datensätzen.

Entsprechend der Anfangsbuchstaben werden diese Operationen als CRUD abgekürzt.

Transaktionen

Eine Transaktion besteht aus einer oder mehreren logisch zusammengehörigen Operationen, die als Einheit ausgeführt werden sollen.

Beispiel: Geldüberweisung - Abbuchung von Konto A und Gutschrift auf Konto B.

  • Update Operation: Geld auf Konto A um Überweisungsbetrag reduzieren
  • Update Operation: Geld auf Konto B um Überweisungsbetrag erhöhen

Kommt es zu Fehlern bei einer Operation, wird die Transaktion insgesamt wieder rückgängig gemacht und der ursprüngliche Zustand wiederhergestellt.

ACID Eigenschaften
  • Atomicity = Atomarität (Ganz oder Garnicht)
    • Alle Operationen einer Transaktion werden vollständig ausgeführt oder rückgängig gemacht (Rollback).
    • Beispiel: Geld, dass von Konto A abgebucht wird, kommt auch sicher auf Konto B an (oder wird nicht abgebucht).
  • Consistency = Konsistenz (Daten bleiben korrekt)
    • Nach einer Transaktion müssen die Daten immer in einem gültigen Zustand sein, der festgelegten Regeln entspricht.
    • Beispiel: Konto wird nicht überzogen, falls dies nicht erlaubt war.
  • Isolation = Isolation (Unabhängigkeit)
    • Parallel ausgeführte Transaktionen beeinflussen sich nicht gegenseitig. Sie müssen zu dem gleichen Ergebnis führen, als wären sie nacheinander ausgeführt worden.
    • Beispiel: Zwei Benutzer buchen einen Kinoplatz. Es muss sichergestellt sein, dass der gleiche Platz nicht mehrfach vergeben wird.
  • Durability = Dauerhaftigkeit (Nichts geht verloren)
    • Erfolgreich abgeschlossene Transaktionen bleiben dauerhaft gespeichert, auch bei Systemausfällen.
    • Beispiel: Eine Bestellung bleibt nach Abschluss sicher erhalten.

Relationale Datenbanken

Grundlagen

Relationale Datenbanken strukturieren Daten in Tabellen. Jede Tabelle besteht aus:

  • Zeilen (Rows): Einzelner Eintrag eines Datensatzes.
  • Spalten (Columns): Eigenschaften, die für jeden Eintrag gespeichert werden sollen.
  • Primärschlüssel (Primary Key): Um auf einzelne Einträge zugreifen zu können, muss es eine eindeutige Kennung geben. Dies kann eine einzelne Eigenschaft, oft eine fortlaufende ID, oder eine Kombination aus Eigenschaften sein.
  • Fremdschlüssel (Foreign Key): Tabellen können über den Primärschlüssel miteinander verbunden sein. Der Primärschlüssel wird dazu als Eigenschaft in eine anderen Tabelle eingefügt.

Normalisierung

Normalisierung reduziert Redundanzen. Wenn der gleiche Wert nicht mehrfach gespeichert wird, verhindert dies ungleiche Angaben zur gleichen Information also Inkonsistenzen.

Beispiel: DM steht in der nachfolgenden Tabelle mehrfach und könnte an einer Stelle als Abkürzung “DM” und an einer anderen ausgeschrieben “Digitale Medien” eingetragen sein.

Die ersten drei von fünf Stufen der Normalisierung umfassen:

  1. Normalform:
    • Spalten enthalten nur Einzelwerte.
    • Beispiel: Name in Vorname, Nachname aufteilen
  2. Normalform:
    • Tabelle bildet nur einen Sachverhalt ab.
    • Alle Eigenschaften hängen vollständig vom Primärschlüssel ab.
    • Beispiel: Student und Veranstaltung in getrennte Tabellen, da die Veranstaltung nicht von der Matrikelnummer abhängt
  3. Normalform:
    • Tabellen sollen nur direkte Abhängigkeiten beinhalten.
    • Nichtschlüssel-Eigenschaften dürfen nicht voneinander abhängig sein.
    • Beispiel: Studiengang und Fakultät sind voneinander abhängig und sollen in eigene Tabellen. MatrikelNr -> Studiengang und Studiengang -> Fakultät 1

SQLite einrichten

Eine SQLite Datenbank besteht aus einer einzelnen Datei:

  • Im Projektordner eine Datei namens dbName.db anlegen (passenden Datenbanknamen wählen)
  • SQLite Plugin in VSCode installieren
    • SQLite Datei im Projekt auswählen zum anzeigen
    • Show and Run Commands -> CTRL + SHIFT + P
    • SQLite: New Query -> SQL Befehle eingeben
    • SQLite: Run Query -> SQLite Datei auswählen -> führt SQL Befehle aus und zeigt Ergebnis an
  • Optional: DB Browser for SQLite herunterladen
    • SQLite Datei öffnen zum betrachten
    • Execute SQL Tab um SQL Befehle einzugeben und auszuführen

  • Datensätze wurden für die Normalisierung auf mehrere Tabellen aufgeteilt
  • Mit Joins können diese zum Auslesen der Daten wieder zu einer Tabelle zusammengeführt werden
  • Ergebnis kann als virtuelle Tabelle gespeichert werden (View)
