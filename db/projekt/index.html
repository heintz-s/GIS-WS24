Projektstruktur

Prinzipiell kann die Struktur eines Projekts frei gewählt werden. Es ist zu empfehlen, die Dateien eines Projekts in einer Ordnerstruktur nach Zugehörigkeit zu unterteilen.

Beispiel für grundlegende Projektstruktur

  • Es ist sinnvoll den Clientseitigen Code (Frontend) für die Darstellung der Webseite im Browser vom Serverseitigen Node.js Code (Backend) zu trennen.
  • Prinzipiell können die Dateien für die Datenbank mit im Projektordner liegen (Database). Bei SQLite handelt es sich nur um eine Datei, die mit im Backend Ordner liegen kann.
  • .gitignore und .vscode erlauben Einstellungen für den Git Upload und projektbezogene VSCode Einstellungen (Erläuterungen siehe nachfolgend).
  • MongoDB
  • SQLite
  • DirectoryProjektName
    • DirectoryBackend
      • server.js
    • DirectoryDatabase
      • Directorydata
        • db
      • DirectoryzipInstall
        • bin
    • DirectoryFrontend
      • Directorycss
        • styles.css
      • Directorysrc
        • headerFooter.js
        • data.js
        • index.js
        • …
      • index.html
    • Directory.vscode
      • settings.json
    • .gitignore
    • …

Um Probleme (mit LiveServer und Git - siehe unten) zu vermeiden kann der Database Ordner mit der MongoDB Installation auch außerhalb des Projektordners angelegt und prinzipiell auch für mehrere Projekte genutzt werden.

  • DirectoryProjektName
    • DirectoryBackend
      • server.js
      • data.db
    • DirectoryFrontend
      • Directorycss
        • styles.css
      • Directorysrc
        • headerFooter.js
        • data.js
        • index.js
        • …
      • index.html
    • Directory.vscode
      • settings.json
    • .gitignore
    • …

Batch Datei erstellen

Es können Dateien mit Befehlen zum Start der DB und des Servers erstellt werden, welche über einen Doppelklick auf die Datei ausgeführt werden können:

  1. Einfache Textdatei erstellen.
  2. Kommandozeilenbefehle in Datei als Text einfügen (siehe unten).
  3. Dateiendung von .txt zu .cmd oder .bat ändern.
Kommandozeilenbefehle für obige Projektstruktur:

  • Dateien startDatabase.cmd (nur für MongoDB) und startBackend.cmd im Projektordner anlegen.

  • startDatabase.cmd

  • startBackend.cmd

Hinweis
  • Backend über node bei Änderungen immer neu starten.
  • Datenbank bei Fehlern ggf. neu starten.

Git Projekte

Git Projekte sollten nur die Dateien mit eigenem Code beinhalten. Installationsdateien (ggf. auch individuelle Daten) sollten daher vom Upload auf Git ausgeschlossen werden, z. B.

  • Die node_modules Dateien, da diese mit npm install einfach nachinstalliert werden können.
  • Bei MongoDB:
    • Die Datenbankdateien (kompletter Database/zipInstall Ordner)
    • Ggf. auch data/db Ordner, so dass jeder Entwickler seinen eigenen Datenstand haben kann
