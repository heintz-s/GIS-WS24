Web API für Datenbankzugriff

Server API

Eine Server Application Programming Interface (API) dient als Schnittstelle, um eine strukturierte Kommunikation zwischen einem Client und einem Server zu ermöglichen. Sie definiert klar, wie Daten gesendet und abgerufen werden können und welche Operationen unterstützt werden. Typischerweise werden Daten über AJAX-Anfragen gesendet und in einer Datenbank gespeichert.

Grundlagen

  • CRUD-Operationen: Für eine Web-App stellt die Server API meist grundlegende Funktionen zum Datenaustausch bereit.
  • Weitere Operationen: Es können weitere Funktionen zur Verfügung gestellt werden, z. B. für mathematische Berechnungen, die auf dem Server stattfinden sollen.
  • Datenformate: Häufig wird JSON als Datenformat genutzt.
  • Flexibilität: Es existieren verschiedene Ansätze zur Implementierung einer API, die sich an den jeweiligen Anwendungsfall anpassen lassen.

Typen von APIs

Individuelle API
  • URLs können individuell definiert werden, um Daten zu empfangen oder abzurufen.
  • Beispiele:
    • /addItem für Hinzufügen eines Objekts
    • /getItem für Abrufen eines Objekts
    • /updateItem für Aktualisieren eines Objekts
    • /deleteItem für Löschen eines Objekts
  • Parameter können in der URL übergeben werden, z. B. eine ID bei GET- und DELETE-Anfragen.
JSON API
  • JSON wird nicht nur für Anwendungsdaten, sondern auch für Anweisungen genutzt.
  • Beispiel:
  • Mögliche Werte für operation sind z. B. add, get, update, delete.
REST API
  • Eine standardisierte Schnittstelle basierend auf den Prinzipien von Roy Fielding (REST = Representational State Transfer).
  • Merkmale:
    • Nutzung von HTTP-Methoden:
      • POST für Create
      • GET für Read
      • PUT für Update
      • DELETE für Delete
    • Ressourcen werden über URLs beschrieben. (Resourcenname Teil des Pfads)
      • GET /student → Alle Studierendendaten abrufen
      • GET /lectures → Alle Veranstaltungsdaten abrufen
    • IDs werden direkt in die URL integriert:
      • GET /items → Alle Items abrufen
      • GET /items/2 → Item mit der ID 2 abrufen
  • REST kann unterschiedlich strikt implementiert werden.
