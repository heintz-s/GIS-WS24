HTML Grundlagen

HTML-Seite

Erstes Beispiel:

  • beliebigen Editor verwenden (z. B. Visual Studio Code)
  • Code schreiben und in einer Datei mit der Endung “.html” speichern (Startseite = index.html)
  • Datei im Browser öffnen
  • HTML-Renderer (auch Browser Engine) interpretiert HTML Code und stellt die Seite dar

HTML = Hypertext Markup Language

  • Auszeichnungssprache zur Beschreibung der Seitenstruktur
  • Design separat durch CSS
  • keine Programmiersprache
  • aktuelle Version HTML5 (seit 2014)

HTML Baumstruktur

  • HTML Elemente müssen als Baum aufgebaut sein (ineinander verschachtelt)
  • im Browser Debugger ansehen (je nach Browser: Rechtsklick -> Untersuchen; F12; …)

HTML Tags

  • “to tag” = markieren, auszeichnen
  • Tags sind mit <> Zeichen umschlossen, z. B. <html lang="de">, </head>, <br/>
  • öffnender Tag: <html lang="de">, <head>, <body>, usw.
  • schließender Tag mit ”/”: </html>, </head>, </body>, usw.
  • Element = öffnendes Tag + Inhalt + schließendes Tag
  • Elemente ohne Inhalt: z. B. <br />, <br>, <meta charset="utf-8" /> 1
  • Details auf W3C Webseite

HTML Attribute

  • zusätzliche Informationen zu einem HTML Element
  • hinter dem Tagnamen im öffnenden Tag gelistet: z. B. <html lang="de">, <meta charset="utf-8" />
  • Syntax: Attributname = “Wert” 2
  • mehrere Attribute durch Leerzeichen getrennt: z. B. <html lang="de" class="xyz">
  • boolesche Attribute (kein Wert, da true wenn angegeben): z. B. <p hidden>

HTML-Seite Grundgerüst im Detail

TagBedeutung

- Sprache sollte angegeben werden (lang="de")5

Viewport (Metadaten):

  • wichtig für “Responsive Web Design” (Anpassung für unterschiedliche Ausgabegeräte)
  • Angabe zu sichtbarem Bereich der Webseite auf einem Gerät
  • device-width = Breite des anzeigenden Gerätes, initial-scale = Zoomfaktor
  • fehlt diese Angabe legt der Browser eine Standardbreite fest, wodurch die Seite auf kleinen Geräten zu klein dargestellt wird
  • weitere Informationen

Webseite mit mehreren Unterseiten

  • mehrere HTML Dateien, die über Hyperlinks (z. B. in Navigationsmenü) verbunden sind
  • z. B. index.html = Startseite, about.html, contact.html, gallery.html, etc.
  • alle in einem Ordner (https://webseite.de/index.html bzw. https://webseite.de und https://webseite.de/about.html, etc.)
  • oder z. B. Seiten in “page” Unterordner (https://webseite.de und https://webseite.de/page/about.html, etc.)
  • gleichbleibende Elemente (header, footer, navigation) wiederholen sich auf den Seiten -> kann mit Code optimiert werden
